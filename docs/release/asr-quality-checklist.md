# ASR 质量发布检查清单

## 目的

本检查清单确保每次发布都满足 ASR 质量、插入质量和隐私保护的基本要求。

## 发布前检查

### 1. 基准测试验证

- [ ] 运行 ASR 回放测试
  ```bash
  npm run replay:asr -- --output tmp/asr-replay.json
  ```

- [ ] 验证基准测试结果
  ```bash
  npm run verify:asr-benchmarks
  ```

- [ ] 检查关键指标
  - [ ] P95 延迟 <= 500ms
  - [ ] 插入成功率 >= 95%
  - [ ] 平均词错误率 (WER) <= 10%
  - [ ] 跳过率 <= 20%

- [ ] 确认基准测试报告状态为 `passed` 或明确说明的 `skipped`
- [ ] 如果回放数据缺失，记录.fixture 语料库不可用的原因

### 2. 插入兼容性测试（冒烟测试）

验证以下应用场景的插入功能：

- [ ] **浏览器文本框** - Safari/Chrome/Firefox
- [ ] **Electron 编辑器** - Notion/Obsidian/VS Code
- [ ] **聊天应用** - 微信/Slack/Messages
- [ ] **文档编辑器** - Pages/Word/Craft

- [ ] 检查直接插入行为
- [ ] 检查替换选中行为（如应用家族支持）
- [ ] 检查回退到复制行为
- [ ] 确认撤销行为用户可理解

- [ ] 记录任何应用特定的降级模式到跨应用插入矩阵

### 3. 敏感应用和隐私检查

- [ ] 确认敏感应用规则覆盖：
  - [ ] 金融应用
  - [ ] 密码管理器
  - [ ] 管理员/认证相关应用

- [ ] 确认以下功能遵守敏感应用策略：
  - [ ] 自动学习
  - [ ] 粘贴监控
  - [ ] 云端路由

- [ ] 确认调试日志脱敏：
  - [ ] 转写文本被脱敏
  - [ ] 剪贴板内容被脱敏
  - [ ] API 密钥被脱敏

- [ ] 确认隐私相关的用户提示与实际运行时行为一致

### 4. 单元测试

```bash
swift test
```

- [ ] 所有测试通过（24/24）
- [ ] 无新增失败测试
- [ ] 覆盖率未显著下降

### 5. 构建验证

```bash
swift build
```

- [ ] 构建成功
- [ ] 无新增警告（Swift 6 并发警告除外）
- [ ] 二进制大小无显著增长

## 回滚标准

出现以下情况应立即回滚：

- [ ] 回放验证返回 `failed`
- [ ] 插入冒烟测试在高价值应用家族中发现回归
- [ ] 隐私或敏感应用策略被绕过
- [ ] 发布资源不完整或质量摘要缺失

## 发布资源检查

- [ ] 发布说明完整
- [ ] 基准测试摘要已生成
- [ ] 跨应用插入矩阵已更新
- [ ] 已知问题已记录
