## 1. 启动与基础状态

### 1.1 应用可启动
步骤：
1. 启动 MouthType
2. 观察是否出现悬浮胶囊和菜单栏入口

预期结果：
- 应用正常启动
- 悬浮胶囊可见
- 菜单栏图标可见
- 无启动即报错

### 1.2 设置窗口可打开
步骤：
1. 从菜单栏或应用菜单打开设置
2. 检查各 tab 是否可切换

预期结果：
- 设置窗口可正常打开
- 所有设置分组可见
- 输入框和按钮不是灰色不可操作状态

## 2. 权限链路

### 2.1 辅助功能权限状态显示
步骤：
1. 打开设置 > 交互 > 权限
2. 查看辅助功能状态文案
3. 点击“打开辅助功能设置”

预期结果：
- 状态显示与系统真实授权一致
- 能跳转到系统对应设置页

### 2.2 输入监控权限状态显示
步骤：
1. 打开设置 > 交互 > 权限
2. 查看输入监控状态文案
3. 点击“打开输入监控设置”

预期结果：
- 状态显示与系统真实授权一致
- 能跳转到系统对应设置页

## 3. 热键与本地听写

### 3.1 热键按下可开始本地录音
前置条件：
- 本地 Whisper 可用
- 输入监控权限已授权

步骤：
1. 选择一个激活热键
2. 按下热键并说一句短语
3. 松开热键

预期结果：
- 悬浮胶囊进入录音态
- 松开后进入处理态
- 文本被粘贴到前台输入框
- 最终回到 idle

### 3.2 热键切换后立即生效
步骤：
1. 在设置中切换激活热键
2. 使用新热键触发听写
3. 验证旧热键不再触发

预期结果：
- 新热键生效
- 旧热键失效
- 不需要重启应用

## 4. 云端 fallback 边界

### 4.1 本地不可用时允许云端流式回退
前置条件：
- Bailian API key 已配置
- 本地 Whisper 不可用（如模型缺失或二进制缺失）

步骤：
1. 按住热键开始说话
2. 松开热键

预期结果：
- 应用进入流式 fallback 路径
- 松开后不会卡住
- 有识别文本时完成粘贴
- 最终回到 idle

### 4.2 本地可用但单次转写失败时不得自动上云
前置条件：
- 本地 Whisper 可用
- Bailian API key 已配置
- 构造一次本地转写失败场景

步骤：
1. 触发一次本地 tap 听写
2. 让本地转写在运行阶段失败

预期结果：
- 应用进入 error
- 不自动走 Bailian one-shot 上传
- 下一次热键仍可重新开始

## 5. 流式 stop 收尾

### 5.1 key-up 后不挂起
前置条件：
- 处于 Bailian 流式 fallback 路径

步骤：
1. 按住热键说一句较长的话
2. 松开热键
3. 观察 3-5 秒

预期结果：
- 不会长期停留在 streaming
- 不会无限等待
- 最终回到 idle 或明确 error

### 5.2 key-up 后尽量保留尾音
前置条件：
- 处于 Bailian 流式 fallback 路径

步骤：
1. 长按热键说一句尾部信息明显的话
2. 在说完最后一个词后立即松开

预期结果：
- 最终文本尽量包含尾部词语
- 不应系统性截断最后半句

## 6. error 恢复能力

### 6.1 本地录音失败后的恢复
步骤：
1. 触发一次录音失败
2. 再次按下热键尝试录音

预期结果：
- 第二次仍可进入正常录音
- 不会永久卡在 error

### 6.2 流式失败后的恢复
步骤：
1. 触发一次流式失败
2. 再次按下热键

预期结果：
- 可以重新开始新的会话
- 不会遗留旧 streaming 文本或音量状态

### 6.3 粘贴失败后的恢复
步骤：
1. 制造一次 paste 失败
2. 再次触发听写

预期结果：
- 后续会话仍可正常开始
- 不会因为上次失败而彻底不可用

## 7. 设置与安全边界

### 7.1 API key 输入体验
步骤：
1. 在设置中为 Bailian 和 AI 分别输入 API key
2. 测试手工输入、粘贴按钮、Command+V

预期结果：
- 可正常输入与粘贴
- UI 为安全输入样式
- 不会明文暴露已输入内容

### 7.2 Bailian endpoint 限制
步骤：
1. 输入受信任 DashScope realtime 地址
2. 输入非受信任 host
3. 输入错误 path

预期结果：
- 仅受信任 host + path 可被接受
- 非法输入不会形成可用 endpoint
